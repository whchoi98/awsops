# 로그인 가이드

AWSops 대시보드는 Amazon Cognito 기반 인증을 사용합니다. 관리자로부터 받은 이메일과 비밀번호로 로그인할 수 있습니다.

## 로그인 절차

### 1. 대시보드 접속

브라우저에서 대시보드 URL에 접속합니다:
```
https://your-domain.com/awsops/
```

### 2. 인증 화면

CloudFront Lambda@Edge가 인증되지 않은 요청을 감지하면 자동으로 Cognito 로그인 페이지로 리다이렉트됩니다.

### 3. 자격 증명 입력

- **이메일**: 관리자가 등록한 이메일 주소
- **비밀번호**: 설정한 비밀번호

:::tip 최초 로그인
최초 로그인 시 임시 비밀번호를 새 비밀번호로 변경해야 합니다. 비밀번호는 다음 조건을 충족해야 합니다:
- 최소 8자 이상
- 대문자, 소문자, 숫자, 특수문자 포함
:::

### 4. 로그인 완료

인증이 성공하면 대시보드 메인 페이지로 이동합니다.

## 로그아웃 (Sign Out)

로그아웃은 사이드바 상단에 위치한 버튼을 통해 수행합니다.

### 로그아웃 위치
1. 화면 왼쪽 사이드바 상단의 **AWSops** 로고 확인
2. 로고 오른쪽에 있는 **로그아웃 아이콘** (문 아이콘) 클릭

### 로그아웃 동작
- 클릭 시 서버에 `POST /api/auth` 요청 전송
- HttpOnly 쿠키가 서버 측에서 안전하게 삭제됨
- 자동으로 로그인 페이지로 리다이렉트

:::info HttpOnly 쿠키
보안을 위해 인증 토큰은 HttpOnly 쿠키에 저장됩니다. 이 쿠키는 JavaScript로 접근할 수 없어 XSS 공격으로부터 보호됩니다. 따라서 로그아웃 시 서버 측에서 쿠키를 삭제하는 방식을 사용합니다.
:::

## 세션 관리

### 세션 유효 시간
- 기본 세션 유효 시간은 Cognito 설정에 따릅니다
- 세션이 만료되면 자동으로 로그인 페이지로 리다이렉트됩니다

### 다중 디바이스
- 동일 계정으로 여러 디바이스에서 동시 로그인 가능
- 각 디바이스별로 독립적인 세션 관리

## 문제 해결

### 로그인이 안 되는 경우

| 증상 | 해결 방법 |
|------|----------|
| 비밀번호 오류 | 비밀번호 재설정 요청 |
| 계정 잠김 | 관리자에게 문의 |
| 페이지가 로드되지 않음 | 브라우저 캐시 삭제 후 재시도 |

### 로그아웃 후에도 접속되는 경우

1. 브라우저 쿠키 삭제
2. 시크릿/프라이빗 모드에서 테스트
3. 다른 브라우저에서 테스트

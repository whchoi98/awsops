# S3

S3(Simple Storage Service) 버킷을 관리하고 보안 상태를 모니터링합니다.

<Screenshot src="/screenshots/storage/s3.png" alt="S3" />

## 주요 기능

### 통계 카드
- **Total Buckets**: 전체 버킷 수
- **Public Buckets**: 퍼블릭 접근이 허용된 버킷 수 (보안 경고)
- **Versioning**: 버저닝이 활성화된 버킷 수
- **Logging**: 액세스 로깅이 설정된 버킷 수

### TreeMap 시각화
리전별로 버킷을 시각적으로 표시:
- **빨간색**: Public 버킷 (주의 필요)
- **녹색**: Versioning 활성화 버킷
- **청록색**: 일반 버킷

버킷 블록 클릭 시 상세 정보 패널로 이동합니다.

### 시각화 차트
- **Buckets by Region**: 리전별 버킷 분포
- **Security Status**: Private/Public/Versioned/Logging 상태 분포

### 필터링
- 검색창: 버킷 이름으로 검색
- 리전 필터: 특정 리전만 조회
- 접근 필터: Public/Private 버킷만 조회

### 상세 패널
버킷 클릭 시 확인 가능한 정보:
- 버킷 이름, 리전, ARN, 생성일
- 보안 설정 (Public Policy, Block ACLs 등)
- 버저닝, 암호화, 라이프사이클 규칙
- S3 접근 권한이 있는 IAM 역할 목록
- 태그 정보

## 사용 방법

### 버킷 목록 조회
1. TreeMap에서 리전별 버킷 분포 확인
2. 테이블에서 상세 목록 조회
3. 필터를 활용하여 원하는 버킷 검색

### 보안 상태 확인
1. Public Buckets 카드에서 퍼블릭 버킷 수 확인
2. TreeMap에서 빨간색 버킷 식별
3. 접근 필터에서 "Public" 선택하여 목록 확인

### IAM 권한 확인
버킷 상세 패널의 "IAM Roles with S3 Access" 섹션에서 해당 버킷에 접근 가능한 IAM 역할을 확인할 수 있습니다.

## 사용 팁

:::tip 퍼블릭 버킷 주의
퍼블릭 버킷은 데이터 유출 위험이 있습니다. 의도하지 않은 퍼블릭 버킷이 있는지 정기적으로 확인하세요. Public Buckets 카드가 빨간색으로 표시되면 즉시 검토가 필요합니다.
:::

:::info 버저닝 권장
중요한 데이터가 저장된 버킷은 버저닝을 활성화하여 실수로 인한 삭제나 덮어쓰기를 방지하세요. 버저닝이 비활성화된 버킷 수는 Versioning 카드에서 확인할 수 있습니다.
:::

## AI 분석 팁

AI 어시스턴트에 다음과 같이 질문해 보세요:

- "퍼블릭 S3 버킷 목록과 보안 위험 분석해줘"
- "버저닝이 비활성화된 버킷은 어떤 것이 있어?"
- "특정 버킷에 접근 가능한 IAM 역할 분석해줘"
- "S3 버킷 암호화 상태 점검해줘"

:::tip Data Gateway
AI 어시스턴트는 Data Gateway를 통해 S3 버킷 분석, 보안 점검, IAM 정책 시뮬레이션 등을 지원합니다. Security Gateway와 연계하여 심층적인 보안 분석도 가능합니다.
:::

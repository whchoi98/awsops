**멀티 어카운트 지원**(PR #6)과 **실시간 Bedrock 스트리밍**을 핵심으로 한 대규모 릴리스입니다. 보안 수정 25건이 반영되었습니다.

<!--truncate-->

### 멀티 어카운트 지원 (PR #6)

- Steampipe Aggregator 패턴: `aws` = 전체 통합, `aws_{id}` = 개별 계정
- `AccountContext` + `AccountSelector`: 계정 전환 드롭다운 (ARIA, 키보드 지원)
- `AccountBadge`: 계정별 컬러 도트 + alias 표시
- `/accounts` 페이지: 계정 추가/삭제/테스트 기능
- `buildSearchPath(accountId)`: 계정별 search_path 전환
- `runCostQueriesPerAccount()`: 계정별 Cost 쿼리 실행 후 병합
- 모든 25개 SQL 쿼리에 `account_id` 컬럼, 35개 페이지에 `useAccountContext()` 적용
- `cross_account.py`: STS AssumeRole (credential 캐싱, ExternalId, audit logging)
- `scripts/11-setup-multi-account.sh`: 자동화 스크립트

### 보안 수정 (25건)

| 심각도 | 항목 |
|--------|------|
| CRITICAL | AssumeRole 감사 로그, ExternalId 필수화 |
| HIGH | Rate Limiting (5 req/min), 입력 검증 강화, execFileSync |
| MEDIUM | Pool exhaustion 방지, 커넥션 파괴 |

### 실시간 Bedrock 스트리밍

- `InvokeModelWithResponseStreamCommand`: 청크 단위 수신
- `streamBedrockToSSE()`: SSE `chunk` 이벤트 전송
- 클라이언트: `streamingContent` 상태로 ReactMarkdown 실시간 렌더링

### 캐시 프리워밍

- `cache-warmer.ts`: 대시보드(23개) + 모니터링(10개) 쿼리 백그라운드 실행
- 4분 주기 자동 갱신 (5분 TTL 만료 전)
- `ensureCacheWarmerStarted()`: lazy-init 패턴

### 고객 로고

- `public/logos/`: 고객 로고 디렉토리
- `customerLogo`, `customerName`, `customerLogoBg` config 지원
- 서버 재시작 없이 config 변경으로 1분 내 반영

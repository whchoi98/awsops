## AgentCore 대시보드 (`/agentcore`)

Bedrock AgentCore Runtime과 8개 Gateway의 상태를 한눈에 파악할 수 있는 전용 대시보드입니다.

<AgentCoreDashboardView />

- **Runtime 상태**: `data/config.json`의 `agentRuntimeArn`으로 상태 조회
- **8 Gateway 카드**: 각 Gateway의 상태, 도구 수, 주요 기능 표시
- **125 도구 전체 목록**: 검색, Gateway별 필터 지원
- **아키텍처 다이어그램**: Runtime → Gateway → Lambda 관계 시각화

:::info API 엔드포인트
`/api/agentcore` (`src/app/api/agentcore/route.ts`)는 `data/config.json`에서 ARN을 읽어 AWS API로 상태를 조회합니다. 하드코딩된 계정 정보가 없어 config만 변경하면 어떤 환경에서도 작동합니다.
:::

## Sign Out 기능

Header에 로그아웃 버튼을 추가하여 Cognito 세션을 안전하게 종료합니다.

| 항목 | Before | After |
|------|--------|-------|
| 로그아웃 | 불가능 (브라우저 닫기만) | Header 로그아웃 버튼 |
| 쿠키 삭제 | `document.cookie` (HttpOnly 접근 불가) | `POST /api/auth` 서버 사이드 삭제 |
| 인증 상태 | 브라우저 캐시에 의존 | Cognito 재인증 강제 |

:::warning HttpOnly 쿠키 삭제
`document.cookie`로는 HttpOnly 쿠키를 삭제할 수 없습니다.
반드시 **서버 사이드 API** (`src/app/api/auth/route.ts`)를 통해 `Set-Cookie: token=; Max-Age=0`을 반환해야 합니다.
:::

## CloudWatch 메트릭 테이블 (4개 서비스)

4개 서비스에 대해 전용 API 라우트와 CloudWatch 메트릭 테이블을 추가했습니다. 모두 동일한 패턴(`cloudwatch get-metric-data` → 프로그레스 바 + 수치)을 사용합니다.

<CloudWatchMetricsFlow />

| 서비스 | API 라우트 | 메트릭 |
|--------|-----------|--------|
| **MSK** | `/api/msk` | 브로커별 CPU, Memory, BytesIn, BytesOut |
| **RDS** | `/api/rds` | 인스턴스별 CPU, Memory, Connections, IOPS |
| **ElastiCache** | `/api/elasticache` | 노드별 CPU, Memory, Network, Connections + Valkey 엔진 카드 |
| **OpenSearch** | `/api/opensearch` | 도메인별 CPU, JVM Memory, Cluster Status, Search Rate |

## Resource Inventory (PR #1)

`/inventory` 페이지에서 18종 리소스 수량 추이를 모니터링합니다.

- **멀티라인 차트**: 리소스별 수량 변화 시계열
- **비용 영향 추정**: 리소스 변화에 따른 예상 비용 표시
- **`src/lib/resource-inventory.ts`**: 스냅샷 기반 (`data/inventory/`), 추가 쿼리 0건
- **Cost Explorer 가용성**: `checkCostAvailability()`로 MSP Payer 자동 판별

## S3 / RDS / ElastiCache 강화

| 서비스 | 추가 기능 |
|--------|-----------|
| **S3** | Bucket Map (TreeMap 시각화), 검색/필터, IAM Roles 표시 |

| **ElastiCache** | Security Groups 탭, CloudWatch 메트릭, Valkey 엔진 지원 |

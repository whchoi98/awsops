클라우드 운영의 도전 과제

Console Hopping

  • EC2 확인 → CloudWatch → VPC → IAM → Cost Explorer
  • 평균 5-7개 콘솔 페이지를 오가며 문제 해결
  • 멀티 어카운트 환경에서는 로그인만 10번

데이터 사일로

  • CloudWatch 메트릭 ≠ Prometheus 메트릭 ≠ 로그 ≠ 트레이스
  • 교차 분석 불가 → 근본 원인 파악 지연

반복적 수작업

  • "이 인스턴스 rightsizing 필요한가?"
  • "미사용 리소스 정리해야 하는데..."
  • 매번 같은 CLI 명령어 반복 실행

보고서 작성 부담

  • Well-Architected Review 수작업
  • FinOps 리포트 매월 수동 작성
  • 2-3일 소요 → 실시간성 부족

AWSops — Single Pane of Glass

Data Layer — Steampipe

  • Steampipe — AWS API를 PostgreSQL 테이블로 변환
  • 380+ AWS Tables — EC2, S3, RDS, Lambda, IAM, VPC 등
  • 60+ K8s Tables — Pod, Deployment, Service, Node
  • pg Pool — CLI 대비 660x 빠른 직접 연결
  • Multi-Account — Aggregator 패턴

AI Engine — Bedrock AgentCore

  • Bedrock Claude — Sonnet 4.6 (분류) + Opus 4.6 (분석)
  • 8 MCP Gateways — 142 Tools, 23 Lambda
  • Code Interpreter — Python 샌드박스

Dashboard — Next.js 14

  • 40 Pages — 컴퓨팅, 네트워크, 스토리지, 보안, 비용, K8s
  • 13 API Routes — Steampipe, AI, CloudWatch 등
  • 18 AI Routes — Sonnet 자동 분류, 멀티 라우트
  • 6 Auto-Collect Agents — EKS/DB/MSK/Idle/Trace/Incident

종합진단 리포트

  • 15 섹션 — Well-Architected 6 Pillar 기반
  • Opus 4.6 — 심층 분석 + PPTX 자동 생성

AWSops가 주는 가치

For DevOps / SRE

  • Console Hopping 제거 — 한 화면에서 모든 리소스
  • 자연어 트러블슈팅 — "VPC Flow Log 분석해줘"
  • 실시간 인시던트 분석 — Prometheus + Loki + Tempo + CloudWatch 교차 분석
  • K9s 스타일 터미널 UI 포함

For FinOps / Management

  • 자동 비용 최적화 — EKS/RDS/MSK rightsizing
  • 유휴 리소스 스캔 — 미사용 EBS, EIP, 중지된 EC2
  • 종합진단 리포트 — PPTX 자동 생성 (15 섹션, 6 Pillar)
  • CIS 컴플라이언스 — v1.5 ~ v4.0 자동 벤치마크

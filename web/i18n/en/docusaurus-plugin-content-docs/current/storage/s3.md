# S3

Manage S3 (Simple Storage Service) buckets and monitor security status.

<Screenshot src="/screenshots/storage/s3.png" alt="S3" />

## Key Features

### Statistics Cards
- **Total Buckets**: Total number of buckets
- **Public Buckets**: Number of buckets with public access enabled (security warning)
- **Versioning**: Number of buckets with versioning enabled
- **Logging**: Number of buckets with access logging configured

### TreeMap Visualization
Visually display buckets by region:
- **Red**: Public buckets (requires attention)
- **Green**: Buckets with versioning enabled
- **Cyan**: Regular buckets

Click on a bucket block to navigate to the detail panel.

### Visualization Charts
- **Buckets by Region**: Bucket distribution by region
- **Security Status**: Distribution of Private/Public/Versioned/Logging status

### Filtering
- Search box: Search by bucket name
- Region filter: View only specific regions
- Access filter: View only Public/Private buckets

### Detail Panel
Information available when clicking on a bucket:
- Bucket name, region, ARN, creation date
- Security settings (Public Policy, Block ACLs, etc.)
- Versioning, encryption, lifecycle rules
- List of IAM roles with S3 access
- Tag information

## How to Use

### View Bucket List
1. Check bucket distribution by region in the TreeMap
2. View detailed list in the table
3. Use filters to search for desired buckets

### Check Security Status
1. Check the number of public buckets in the Public Buckets card
2. Identify red buckets in the TreeMap
3. Select "Public" in the access filter to view the list

### Check IAM Permissions
In the "IAM Roles with S3 Access" section of the bucket detail panel, you can view IAM roles that have access to the bucket.

## Tips

:::tip Public Bucket Warning
Public buckets risk data exposure. Regularly check for unintentionally public buckets. If the Public Buckets card is displayed in red, immediate review is required.
:::

:::info Versioning Recommended
Enable versioning on buckets storing important data to prevent accidental deletion or overwriting. The number of buckets without versioning can be checked in the Versioning card.
:::

## AI Analysis Tips

Try asking the AI assistant:

- "Analyze the list of public S3 buckets and security risks"
- "Which buckets have versioning disabled?"
- "Analyze IAM roles with access to a specific bucket"
- "Check S3 bucket encryption status"

:::tip Data Gateway
The AI assistant supports S3 bucket analysis, security checks, and IAM policy simulation through the Data Gateway. In-depth security analysis is also possible in conjunction with the Security Gateway.
:::

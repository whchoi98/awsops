A major release featuring **multi-account support** (PR #6) and **real-time Bedrock streaming** as core features. Includes 25 security fixes.

<!--truncate-->

### Multi-Account Support (PR #6)

- Steampipe Aggregator pattern: `aws` = all accounts merged, `aws_{id}` = individual account
- `AccountContext` + `AccountSelector`: account switching dropdown (ARIA, keyboard support)
- `AccountBadge`: per-account color dot + alias display
- `/accounts` page: add/remove/test account functionality
- `buildSearchPath(accountId)`: per-account search_path switching
- `runCostQueriesPerAccount()`: execute Cost queries per account then merge
- All 25 SQL queries include `account_id` column, 35 pages use `useAccountContext()`
- `cross_account.py`: STS AssumeRole (credential caching, ExternalId, audit logging)
- `scripts/11-setup-multi-account.sh`: automation script

### Security Fixes (25 items)

| Severity | Items |
|----------|-------|
| CRITICAL | AssumeRole audit logging, ExternalId enforcement |
| HIGH | Rate Limiting (5 req/min), input validation hardening, execFileSync |
| MEDIUM | Pool exhaustion prevention, connection destruction |

### Real-time Bedrock Streaming

- `InvokeModelWithResponseStreamCommand`: chunk-based reception
- `streamBedrockToSSE()`: SSE `chunk` event transmission
- Client: real-time ReactMarkdown rendering with `streamingContent` state

### Cache Pre-warming

- `cache-warmer.ts`: background execution of dashboard (23) + monitoring (10) queries
- 4-minute auto-refresh cycle (before 5-min TTL expiration)
- `ensureCacheWarmerStarted()`: lazy-init pattern

### Customer Logo

- `public/logos/`: customer logo directory
- `customerLogo`, `customerName`, `customerLogoBg` config support
- Reflects config changes within 1 minute without server restart

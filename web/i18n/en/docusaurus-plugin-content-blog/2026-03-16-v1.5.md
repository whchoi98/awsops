**AgentCore Memory Store** introduced for 365-day per-user conversation history retention. New EBS, MSK, OpenSearch pages and ECS/EKS container cost analysis added.

<!--truncate-->

### AgentCore Memory Store

- `06e-setup-agentcore-memory.sh` — Memory creation (365-day retention)
- Automatic conversation saving: question/summary/route/tools/response time
- Per-user isolation: email extracted from Cognito JWT
- AgentCore page: Call statistics (6 StatsCards) + conversation history search
- AI page bottom: Conversation history toggle panel

### New Pages (4)

- **EBS** (`/ebs`): Volumes/snapshots, encryption, EC2 attachments, idle volume detection
- **MSK** (`/msk`): Kafka clusters, broker nodes (CPU/Memory/Network), KRaft
- **OpenSearch** (`/opensearch`): Domains, encryption (N2N/At-Rest), VPC, cluster configuration
- **Resource Inventory** (`/inventory`): Quantity trends for 18 resource types

### Container Cost Analysis

- `/container-cost`: ECS Fargate cost (Container Insights + price calculation)
- `/eks-container-cost`: EKS Pod cost (OpenCost API + request-based fallback)
- `06f-setup-opencost.sh`: Prometheus + OpenCost installation

### AI Routing Improvements

- AgentCore configuration externalized: ARNs read from `data/config.json`
- Tool usage display: MCP tools inferred via keyword matching, shown as UI badges
- Multi-route failure falls back to Bedrock Direct, timeout increased from 60s to 90s

### Security Enhancements

- Sign Out: HttpOnly cookie deletion via server-side API
- Removed 3 hardcoded values (password, interpreter ID, gateway URLs)
